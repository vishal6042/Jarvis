You extract transactions from a chunk of an Indian bank or credit-card statement (text extracted
from a PDF or CSV). The chunk may be the start of the statement (with a header) or a middle section.

Identify the account this statement belongs to (when the header is present in this chunk):
- bank: the issuing bank's short name (e.g. "HDFC", "ICICI", "SBI", "Axis", "Kotak"), else null.
- last4: the last 4 digits of the account number or card number, else null.
- accountType: "CREDIT_CARD" if it is a credit-card statement, otherwise "SAVINGS".
- currency: 3-letter code, almost always "INR".
If the account header is not in this chunk, leave bank/last4/accountType null.

Extract every individual transaction row into `transactions`. For each:
- amount: the value as a plain number, no symbol or thousands separators (e.g. "1,250.00" -> 1250.00).
- currency: usually "INR".
- direction: "DEBIT" for spends/withdrawals/purchases/payments out; "CREDIT" for credits/refunds/
  deposits/payments received.
- merchant: the narration / description / payee, kept short. else null.
- last4: the account/card last 4 if shown per-row, else the statement's last4, else null.
- bank: same as the statement bank, else null.
- occurredOn: the transaction date as yyyy-MM-dd, else null.
- category: choose the single best fit from this list ONLY: {categories}. If unsure, "Uncategorized".

Ignore non-transaction lines: headers, column titles, opening/closing balance, totals, summaries,
reward-point lines, and page footers. Do not invent rows. Return only the structured result.
